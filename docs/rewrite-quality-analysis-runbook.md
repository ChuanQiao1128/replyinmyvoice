Scope: How the owner generates, reads, stores, and acts on internal Rewrite Quality Analysis reports for `replyinmyvoice.com`.

## 1. Purpose

The Rewrite Quality Analysis report is an internal operating tool. It answers five owner questions:

1. Are successful rewrites preserving facts and improving the writing signal?
2. Which failure reasons are most common?
3. Which strategy version performs best for quality, cost, and latency?
4. Are provider costs still compatible with the live rewrite packs and Pro/API pricing?
5. Should the current strategy be repaired, canaried, rolled back, or left alone?

Do not use this report as public marketing copy. It is evidence for product, support, cost, and rollout decisions.

## 2. Inputs And Outputs

The report script reads aggregate telemetry from:

| Source | Purpose |
|---|---|
| `RewriteCostLog` | Request status, strategy version, writing-signal values, duration, costs, candidate counts, escalation flag, and primary error label. |
| `RewriteProviderCall` | Provider, role, model, token or character counts, provider latency, provider cost estimate, success flag, and provider error label. |

The script writes:

| Artifact | Default path | Commit policy |
|---|---|---|
| Markdown report | `docs/rewrite-quality-analysis-report.md` | Commit only after the privacy checklist in Section 7 passes. |
| Summary CSV | `exports/rewrite-quality-summary.csv` | Commit only after review; keep local if it contains sensitive launch volume or cost timing. |
| Charts | `exports/charts/*.png` | Commit only after review; safe only when generated by the script and visually checked. |

Never commit database URLs, raw SQL dumps, provider dashboard screenshots, raw user messages, rough drafts, final rewrites, user identifiers, API keys, or `.env.local`.

## 3. Generate A Report

Use a read-only telemetry database connection. Prefer a dedicated read-only URL when available.

```bash
export ANALYTICS_DATABASE_URL="postgresql://..."

python3 scripts/analyze_rewrite_quality.py \
  --since 2026-05-01 \
  --until 2026-05-22 \
  --database-url-env ANALYTICS_DATABASE_URL \
  --output docs/rewrite-quality-analysis-report.md \
  --summary-csv exports/rewrite-quality-summary.csv \
  --charts-dir exports/charts
```

The script prints only output paths on success. If it reports missing schema columns, finish the M5 telemetry issues before using the report for launch decisions.

For local fixture verification, run the unit test suite instead of pointing the script at production:

```bash
npm run test -- tests/unit/analyze-rewrite-quality.test.ts
```

## 4. Read The Key Metrics

| Metric | Meaning | Owner interpretation |
|---|---|---|
| Total requests | Matching rewrite attempts in the selected period. | Check that the period has enough volume before making strategy decisions. |
| Success rate | Share of requests that produced a chargeable successful rewrite. | Low success rate means customers may see too many quality failures or server failures. |
| Quality-failure rate | Share stopped by quality gates rather than server failure. | A high rate protects users from bad output but points to strategy, prompt, or gate tuning work. |
| Server failures | Requests that failed because infrastructure or providers failed. | Treat repeated server failures as reliability work, not prompt work. |
| Average signal drop | Average draft signal minus rewrite signal for measured rows. | Higher is better, but only when facts and tone are still preserved. |
| Below 50% signal rate | Share of measured rewrites under the internal signal threshold. | Useful trend metric; not a user-facing guarantee. |
| No-regression rate | Share of measured rows where the rewrite signal did not get worse. | Anything below 95% deserves review of selection and quality gates. |
| Signal unavailable rows | Rows excluded from signal averages because a signal was missing. | Do not claim a launch-quality signal result from a run with unavailable signal rows. |
| Fact-failure count | Requests labeled `fact_check_failed`. | Any spike is a priority for fact-gate hardening and regression tests. |
| Cost per successful rewrite | Estimated provider cost divided by successful rewrites. | Compare against the subscription margin and quota budget. |
| Duration p50 / p95 | Median and high-end request latency. | High p95 usually means retries, escalation, or provider latency need attention. |
| Escalation rate | Share of requests that used the expensive repair or escalation path. | High rate may be justified for hard cases but can hurt cost and latency. |
| Average internal strategies | Average number of bounded strategies attempted per request. | Rising values show the router is working harder; verify quality gains justify it. |

## 5. Read Failure Labels

Start with the `Failure Labels` section of the report.

| Label pattern | Likely action |
|---|---|
| `signal_unavailable` | Improve Sapling/provider retry behavior, timeout handling, and no-charge handling. |
| `naturalness_gate_failed` | Repair prompt/style strategy, strategy routing, or candidate selection. |
| `fact_check_failed` | Harden fact extraction, add regression cases, and inspect the fact-gate rule that rejected the candidate. |
| `reviewer_threshold_failed` | Review structured reviewer thresholds and whether the scenario needs a more specific strategy. |
| `server_failed` | Investigate runtime errors, database writes, or provider call handling before changing prompts. |
| `<provider>:<error>` | Check provider health, retry policy, fallback route, and cost impact for that provider role. |

Do not treat all failures as rewrite-quality failures. A provider outage, missing signal, and fact-gate rejection drive different fixes.

## 6. Strategy Decisions

Use the `Strategy Version Comparison` table before promoting or rolling back a strategy.

Promote or expand a canary only when the new strategy has:

- equal or better success rate;
- equal or better average signal drop on measured rows;
- no drop in no-regression behavior;
- no fact-failure spike;
- acceptable cost per successful rewrite;
- acceptable duration p95 and escalation rate.

Repair the prompt or strategy router when:

- quality-failure rate rises above 10%;
- `naturalness_gate_failed` or `reviewer_threshold_failed` dominates failures;
- average signal drop falls while cost or strategy attempts increase;
- a scenario repeatedly needs escalation before success.

Harden fact gates when:

- `fact_check_failed` appears in a launch window;
- support reports changed facts;
- a strategy version succeeds on signal but loses names, dates, money, deadlines, policy constraints, or no-change-without-confirmation constraints.

Improve provider retry or fallback when:

- `signal_unavailable` appears in a measured launch report;
- provider error labels cluster around one role or model;
- server failures increase without a product code change.

Tune budget when:

- cost per successful rewrite trends above the current margin target;
- escalation rate rises without a matching quality gain;
- average internal strategies climbs for routine cases.

Roll back or pause canary traffic when:

- the canary trails control strategy signal drop by 3 points or more over the configured window;
- fact failures increase;
- p95 latency or cost per success becomes unacceptable;
- customer support receives multiple same-pattern rewrite complaints.

## 7. Privacy And Commit Checklist

Before committing any generated report, CSV, or chart:

1. Open the Markdown report and confirm it contains only aggregate metrics, labels, strategy versions, providers, models, dates, and chart paths.
2. Confirm it does not include raw user messages, rough drafts, final rewrites, names from user text, email addresses, user IDs, request IDs, database URLs, or secrets.
3. Open the CSV and confirm it contains only `metric`, `segment`, `value`, and `period` rows.
4. Open each PNG chart and confirm it shows only aggregate trends, not screenshots from dashboards or databases.
5. If the run used real production data, decide whether launch volume, cost, or strategy-version timing is sensitive. Keep artifacts local if publishing those aggregates would hurt the business.
6. Do not paste terminal output if it includes environment variable values or connection strings.

Safe to commit by default:

- this runbook;
- `docs/rewrite-quality-analysis-spec.md`;
- analyzer source and tests;
- fixture-generated reports that are clearly synthetic.

Requires owner review before commit:

- reports, CSV files, and charts generated from real telemetry.

Never safe to commit:

- `.env.local`;
- database URLs;
- provider keys;
- raw database exports;
- raw rewrite content;
- screenshots that show customer or account data.

## 8. Owner Review Cadence

During launch week:

- run the report daily for the previous 24 hours;
- keep artifacts local unless they are needed for a launch-readiness record;
- record decisions in `plans/decisions-log.md` without raw customer content.

After launch stabilizes:

- run weekly for the previous 7 days;
- compare strategy versions before any canary ramp;
- use the report alongside `docs/learningops-runbook.md` and `docs/rewrite-strategy-memory.md`.

Before advertising quality claims:

- run a current measured report;
- confirm no unavailable signal rows in the evaluated window;
- confirm known failure samples are fixed or documented as non-launch blockers;
- keep the claim internal unless the owner explicitly approves public wording.

## 9. Troubleshooting

| Symptom | Next step |
|---|---|
| Script says the database URL is missing | Set the configured environment variable. Prefer `ANALYTICS_DATABASE_URL` for read-only production analysis. |
| Script says schema is not ready | Check that M5-002/M5-004 telemetry columns exist and migrations are deployed. |
| Report has zero rows | Confirm `--since` and `--until`; then verify production telemetry is writing `RewriteCostLog` rows. |
| Signal averages are `n/a` | Inspect Sapling availability and `rewriteAiLikePercent` / `draftAiLikePercent` persistence. |
| Cost per success is `n/a` | Confirm there are successful rows in the period. |
| Charts are blank | Empty datasets still generate placeholder charts; verify the period and row count. |
| Provider cost looks wrong | Check model pricing configuration and `RewriteProviderCall` token or character counts. |
