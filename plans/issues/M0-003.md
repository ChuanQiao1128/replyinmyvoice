# M0-003: Discard or stash remaining dirty files

**Milestone**: M0-Stabilize
**Est**: ≤15 min
**Depends on**: M0-001 (triage), can run before or after M0-002

## TASK
Per `plans/wip-triage.md`'s "Discard" and "Stash" sections, clean the working tree so `git status` is empty (except for already-committed branches from M0-002).

## CONTEXT
- Triage decisions live in `plans/wip-triage.md`
- "Discard" = `git restore <file>` (lose the changes)
- "Stash" = `git stash push -m "<descriptive name>" -- <files>` (save for later)

## CONSTRAINTS
- Each stash gets a descriptive name with restore criteria from the triage report
- Do not stash files marked "discard"
- Do not restore files marked "stash"
- Do not touch files already committed by M0-002

## CHANGES REQUIRED
1. For each file in triage "Discard":
   - `git restore <file>` (or `git checkout -- <file>` for older git)
2. For each grouped "Stash" set:
   - `git stash push -m "wip: <theme> — restore when <criterion>" -- <file1> <file2> ...`
3. Confirm `git status --short` is empty
4. List active stashes with `git stash list` and append to `plans/wip-triage.md` under a new "Active stashes (post M0-003)" section

## ACCEPTANCE
- `git status --short` outputs nothing (or only your branch tracking)
- `git stash list` shows the new stashes with descriptive names
- `plans/wip-triage.md` has a final section listing active stashes for future reference

## DO NOT
- `git stash clear` or `git stash drop`
- Delete any file from disk (only `git restore`)
- Push the stash anywhere (stash is local-only)
- Commit, push, or deploy

## On completion
Comment with the list of discarded files (count) and active stashes (count + names). Close the issue.
